# Codex Package Docs

This directory contains design notes for the Codex package.

Docs here are not installed into Codex by `basecamp sync codex`. They are the
working map for deciding what belongs in `codex/` and how the package should use
Codex-native surfaces such as skills, MCP servers, hooks, custom agents,
settings, and project instructions.

## Documents

- `package-design.md` describes the package architecture and the boundary
  between skills, MCP tools, hooks, custom agents, and settings.
- `migration.md` describes the cutover plan for making the Codex package the
  primary home for Codex-facing engineering workflows.
